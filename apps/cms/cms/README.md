# DIRECTUS - headless cms 
 1 - документация - https://docs.directus.io/

## Установка:

1. устанавливаем зависимости
    ```shell
    pnpm i
    ```
2. Устанавливаем себе клиент докера, если он не установлен - https://www.docker.com/products/docker-desktop/

## Запуск:

1. Запуск либо через расширени IDE или через скрипт 
    ```shell
    pnpm start
    ```

## Работа:

1. После запуска цмс локально, вы можетее создавать модели через UI самой цмски
2. После создания сущностей, закрывающих наши потребности, необходимо сгенерировать snapshot бд, он необходим для того чтобы поддерживать консистентность базы данных на прод окружении
    ```shell
    pnpm generate-snapshot
    ``` 
3. Комитим изменения вместе со сгенерированой схемой

## Миграции

1. При создании скрипта миграции перед датой всегда добавляем символ "U", это нужно чтобы исключить вероятность конфликтов с миграциями которые добавляет команда "directus"
